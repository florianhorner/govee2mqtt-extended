### 🐛 Bug Fixes

- Fix: replace .expect() panics with graceful error handling in MQTT entities

Replace all `.expect("device to exist")` calls in notify_state() with
`let-else` patterns that log a warning and return Ok(()) instead of
panicking. This prevents the entire service from crashing when a device
is temporarily missing from state during re-enumeration or startup.

Also fix the exit code in spawn_hass_integration: when run_mqtt_loop
exits unexpectedly, the process now calls exit(1) instead of exit(0),
so Home Assistant's supervisor correctly detects the failure and
restarts the addon.

Affected files:
- climate.rs, humidifier.rs, light.rs, number.rs, select.rs (x2),
  sensor.rs (x2), switch.rs: .expect() → let-else + log::warn
- hass.rs: exit(0) → exit(1), log::info → log::error
